Cascading Style Sheets (CSS)

  • Enables separation of content and presentation
  • Reduces the amount of effort required to style a document or web site
(example)

History of CSS

  • Style sheets have existed in some form since the inception of the Internet
    • CSS was closely related to the development timeline of HTML
  • Limited and patchy adoption has led to inconsistencies between browser support
    • Which still exists today
  • CSS "hacks":
    <!--[if IE 6]> <p>You are using Internet Explorer 6.</p> <![endif]--> <!--[if !IE]> <p>You are not using Internet Explorer.</p> <!--[endif]-->

Advantages of CSS

  • Separates content and structure
    • De-couple each part of the site to what it does best
  • Makes sites "maintainable"
    • Change one tag and all in the document are changed
  • Bandwidth
    • Load the style reference into the cache once only
  • Accessibility
    • Aids the ease of useful semantic structure in a site

The CSS Box Model

Basic Operation

Styles are specified And they are passed down ("cascaded") 


    selector {
        attribute: value;
    }

Simple example


                    h1 {
                    text-align : center;
                    }

Selectors: id and class

  • id identifies a unique element in a document defined in CSS with # (hash) #c100 {styles}

    . . .

  • class identifies several elements belonging to the same group or class defined in CSS with . (dot) .city {styles} OR p.city {styles}

    . . .

  • If element name is not included, style is applied to all elements that use that class

Selector types

    Universal selector: all elements *{color: black} Tag selector: HTML elements e.g.

    h1 {color:black} Class selector: elements with the specified class  e.g.

     .aClass {color:black} h1.aClass {color:black} id selector: Single/Unique element identified by an id  e.g.
     #pageTitle {color:black}

Pseudo Classes

    Syntax selector:pseudo-class {property: value} Anchor pseudo-class ("a") a:link {color: #ff0000} /* unvisited link */ a:visited {color: #00ff00} /* visited link */ a:hover {color: #ff00ff} /* mouse over link */ a:active {color: #0000ff} /* selected link */ Class and Pseudo-class a.red:visited {color: #ff0000} CSS Syntax

Advanced selectors

  • Descendant selector (space in between selectors): div#bigContainer .aClass {property: value}
  • Child selector (>): div.mainText > .aClass {property: value}
  • Adjacent sibling (+): p + div#bigContainer {property: value}
  • First-child pseudo-sibling div > p:first-child {property: value}

Three Methods of Styling XHTML

  • Inline <h1 style="text-align: center">Page Title</h2>
  • Embedded <head> <style> h1 {text-align: center} </style> </head> <body> <h2>Page Title</h2> </body>
  • Imported <link type="text/css" href="my_styles.css" rel="stylesheet"> OR… @import url(mystyles.css)

Order of Application

Colour

  • Expressing Color standard CSS colors red, blue, black, purple … rgb decimal rgb (255, 0, 0) is ‘red' rgb (0, 128, 128) is ‘teal' rgb hexidecimal #ff0000 is ‘red' #008080 is ‘teal' #fffff0 is ‘ivory'
  • Color wheels/charts readily available… Online Photoshop Elements

Fonts

  • font-family: Generic fonts serif, sans-serif, monospace, cursive, fantasy,... Specific fonts Arial, Helvetica, Times, Garamond,... Generic fonts allows the OS to chose the best match, but may not render as you expect Specific fonts provide a more accurate idea of the display, but the OS may not have the font It is common to specify more than one font, from specific to generic – to cover your bases…

Positioning, Border, Size

  • Positioning absolute/relative: absolute position within block/relative from default location float: allow text to wrap round element clear: do not allow text wrapping (e.g. {clear: right;})
  • Border borders, margins, padding e.g. All have variants with ‘top', ‘bottom', ‘right', ‘left': {padding-left:10pt} {margin-bottom:5px}
  • Size (width and height) Absolute: px (smallest display element), cm/mm/in/... etc Relative: % (percent of parent) em (width M in default font) ex (height x in default font)

Media

  • Specify CSS for audio: @import url("voice.css") aural;
  • Larger font for print: @media print { body { font-size: 12pt }}
  • Smaller font for screen: @media screen { body { font-size: 10pt }}
  • And the HTML looks like: <link rel="stylesheet" type="text/css" media="print, handheld" href="foo.css">

Inheritance

An xhtml element that lies within another xhtml element inherits some of the styles of its parent:

Example: <body> <p>This is <strong>very</strong> important</p> </body> Some styles allocated to <body> are inherited by <p> and <strong> Applicable to styles like color and font.

Do this with a combination of JavaScript and CSS (the user agent is the key variable):

    function() {

    }

var cssFile = "specific_browser_name.css";

    cssFile = "mobile_browser_name.css"
}

Limitations of CSS

  • Can't target specific text without markup tags With the exception of the :first-letter pseudo-element
  • Can't name rules
  • No expressions
  • Very easy to overwrite other rules in a complex CSS file
