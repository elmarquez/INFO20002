Architecture of the World Wide Web

  • Documents encoded in a hypertext format (HTML)
  • Web servers that make resources accessible by a standard scheme (HTTP), on a well known port (80)
  • The location of a resource on a server specified by a Uniform Resource Locator (URL)
  • Browsers use URLs to retrieve resources from servers and render them for the user
  • Documents are connected by one way links
  • Works on top of the existing Internet

HTML is an Evolving Standard

1995 HTML 2.0 The original HTML is mostly structural markup (<h1>,<h2>,<p>), and is not intended for formatting a document.
1997 HTML 3.2 Stylistic markup like <font> and color attributes were added. Mix of content and presentation: content chaos, the era of colorful but messy web pages.
1998, 1999 HTML 4.0, 4.01 Formatting is removed from the document and stored in a separate style sheet (CSS). Presentation is separated from the document structure.
2000 XHTML 1.0 Structure is further enforced. Stricter and cleaner document. HTML as XML document.
2008 HTML 5 New elements and attributes for modern Web sites, semantic markup, support for multimedia technologies.

HTML vs XHTML

XHTML ... is an XML document ... has stronger structural support ... separates content and presentation ... allows easier content management ... allows easier search engine optimisation ... can be validated * but so does HTML 4...

Well Formed XML

  • Elements must be properly closed with an end tag X <p>This is a para.<p>This is another para. ✓ <p>This is a para.</p><p>This is another para.</p>
  • Empty elements must also be properly closed X <br> ✓ <br /> ✓ <br /> <hr /> <img src="an_image.jpg" />
  • Correct ordering of nested elements X <em><strong>Something is wrong</em></strong> ✓ <em><strong>Something is good</strong></em>

Well Formed XML

  • All attribute values must be properly quoted
    • X <td colspan=3>
    • ✓ <td colspan="3">
  • Attribute minimization is not allowed
    • X <select multiple>…</select>
    • ✓ <select multiple="multiple">… </select>
  • Use appropriate entities
    • X <title>Hansel & Gretal</title>
    • ✓ <title>Hansel & Gretal</title>
    • ✓ <a href="page.php?id=about&style=5">About Us</a>

Document Object Model

Three Broad Types of Markup

  • Structural - Identifies the structure of the document
  • Stylistic - Changes the appearance of text within the document
  • Semantic - Identifies specific types of information within a document

Structural Markup

  • Paragraphs: <p>
  • Headings: <h1>, <h2> <h3>, <h4>, <h5>, <h6>
  • Articles and sections: <article>, <section>
  • Generic containers of content: <div> and <span>

Stylistic Markup

  • Bold text: <em>, <strong>
  • Italic text: <i>

Semantic Markup

  • Tables
  • Forms
  • Images - img
  • <article> <aside> <details> <figcaption> <figure> <footer> <header> <main> <mark> <nav> <section> <summary> <time>

Block and Inline Elements

Block elements expand naturally and naturally drop below other elements Generally considered structural Inline elements can be thought of as a "box that acts like text" Flows with text and is generally considered presentational Block can go inside block Inline can go inside block Block cannot go inside inline
DIV and SPAN Elements <span> element is used for inline markup: <p>I am reading <span class="book">Harry Potter</span></p> … and can only enclose inline elements: text and <img> <div> element is used for block markup: <div class="sale_items"> <p>This week's sale items are:</p> <ul> <li>Pink Lady apples</li> <li>Bananas</li> </ul> </div> … and can enclose any inline or block elements: <p>, <blockquote>, <pre>, <h1>…<h6>, <div>
